# 模拟题专题（CSP）

## 什么是模拟题？在 CSP 中的出现频率

**模拟题**指：题目用自然语言或规则描述一系列操作，选手只需**严格按题意实现步骤**，通常不依赖高深定理，而依赖**细心、边界与代码组织**。在 CSP 中，**T1、T2 大量为模拟或简单算法**；T3 也常出现「大模拟 + 少量优化/数据结构」的混合形式。

可以理解为：**把题面翻译成程序**，核心是「不曲解、不漏条件、不越界」。

---

## 常见类型

### 1. 数组操作

在一维数组上统计、筛选、维护前缀/差分思想（即使不用显式差分数组）。

- 典型：折点、峰值、滑动窗口式扫描。

### 2. 矩阵变换

二维数组表示网格或图像，涉及旋转、翻转、坐标映射。

- 关键：搞清楚「行、列与输出顺序」，先写 `(i, j) → (i', j')` 的对应关系。

### 3. 规则模拟

多步骤流程：循环或递归按规则更新状态，直到终止条件。

- 建议：把「一步操作」封装成函数，主循环只负责调用与判断结束。

### 4. 日期与时间

年月日推算、星期、闰年判断（若考纲涉及）。

- 关键：闰年规则、每月天数表；可用「累计天数」化简比较。

---

## 解题技巧

### 仔细读题

- 圈出：**数据范围**、**输出格式**（空格、换行、是否取模）、**多测与否**。
- 对「保证」「恰好」「至多」等词敏感。

### 边界条件

- 空集、单元素、`n=1`、最大/最小坐标、首尾格。
- 矩阵第一行/最后一行与角点。

### 代码组织

- 先写输入解析，再写核心逻辑，最后统一输出。
- 魔法数字尽量用命名常量，便于对照题意修改。

---

## 常见坑点

| 坑点 | 说明 | 应对 |
|------|------|------|
| **整数溢出** | 乘法、累加超 `int` | 中间量用 `long long`，注意乘法顺序 |
| **数组越界** | `i+1`、`i-1`、邻域扩展 | 先判断边界再访问 |
| **Off-by-one** | `[0,n)` 与 `[1,n]`、区间闭开 | 用纸写小例子对齐下标 |
| **浮点误差** | 除法比较相等 | 题意允许时用整数化或 eps；或按题面要求输出位数 |

## 示例代码片段

### 安全邻域访问（四方向）

```cpp
const int dx[] = {-1, 1, 0, 0};
const int dy[] = {0, 0, -1, 1};

bool inside(int x, int y, int n, int m) {
    return x >= 0 && x < n && y >= 0 && y < m;
}

// for (int d = 0; d < 4; ++d) {
//     int nx = x + dx[d], ny = y + dy[d];
//     if (!inside(nx, ny, n, m)) continue;
//     ...
// }
```

### 用 `long long` 防累加溢出

```cpp
long long sum = 0;
for (int v : a) {
    sum += static_cast<long long>(v);
}
```

### 一步一函数（规则模拟骨架）

```cpp
void step(State& s) {
    // 严格按题意更新 s
}

int main() {
    State s = read_input();
    while (!done(s)) step(s);
    print_answer(s);
    return 0;
}
```

## 自测与调试流程（推荐）

1. **样例**：程序跑通题面样例后，再自编「最小」「最大」「全相同」三类小数据。  
2. **对拍**（学有余力）：用朴素程序或 Python 脚本生成随机小数据比对输出。  
3. **静态检查**：循环上界是否 `< n` 还是 `≤ n`；是否在修改数组时又读了旧值。  

---

## 多组输入与清空

若题目含多测，注意每次测试前 **重置 vector、计数器、标记数组**。避免把上一组答案带入下一组。

```cpp
int T;
cin >> T;
while (T--) {
    int n;
    cin >> n;
    vector<int> a(n);
    for (int& x : a) cin >> x;
    // solve(a)
}
```

## 坐标与方向：八邻域扩展（示意）

部分网格题需八连通，方向数组可写为：

```cpp
const int dx8[] = {-1,-1,-1,0,0,1,1,1};
const int dy8[] = {-1,0,1,-1,1,-1,0,1};
```

使用前仍应用 `inside(nx, ny, n, m)` 做边界保护。

## 日期模拟小工具思路

将「某月某日」转为年内第几天，便于比较先后：

- 预存 `month_days[12]`，注意闰年二月为 29 天。  
- `is_leap(y)`：`((y%4==0 && y%100!=0) || (y%400==0))`（按格里高利历题意使用）。

## 与排序、前缀的结合

模拟题常在外层套一层 **排序**（按时间、位置）后再模拟；或维护 **前缀和** 加速区间查询。遇到「区间和频繁查询」且 `n` 大时，应想到树状数组/线段树（见 `topics/data-structures.md`），而非每次暴力累加。

---

## 交互题与文件读入（若赛制涉及）

部分训练平台有 **交互式** 题目：须严格按协议 `flush` 输出。CSP 笔试以文件读写为主，但若练习遇到交互题，注意 **每次询问后立即刷新输出**，并避免死等。

---

## 大数组与栈空间

局部开 `int a[2000000]` 可能导致 **栈溢出**；大数组应使用 `vector` 或静态全局数组（注意多测时清空代价）。二维数组 `vector<vector<int>>` 注意 `reserve` 与 `resize` 的初始化。

---

## 输出格式：空格与换行

- 行末是否允许多余空格：多数 OJ 忽略行末空格，但 **建议按题面格式** 输出。  
- 最后一组数据后是否换行：通常无影响，保持风格一致即可。

---

## 伪代码到代码的翻译表

| 伪代码 | C++ 注意点 |
|--------|------------|
| `for i in 1..n` | 决定用 `for (int i = 1; i <= n; ++i)` 还是 `0..n-1` |
| `取模` | 负数先 `+ MOD` |
| `排序稳定` | `stable_sort` 或显式保留原下标 `pair<key, index>` |

---

## 模拟 + 优先队列

事件驱动模拟（时间戳、操作队列）常配合 `priority_queue` 取 **下一个最早发生的事件**。注意同一时刻多事件的处理顺序是否与题意一致，需要时 **二级排序键**（先类型后时间）。

---

## 精度与整数化

几何或平均值题若要求 **四舍五入到小数**，优先用题给公式化为 **整数分子分母** 比较，避免浮点误差；输出再用 `printf` 控制位数。

---

## 代码长度控制

模拟题容易写长。建议：

- 结构体封装状态，重载 `operator==` 便于调试对比。  
- 复杂条件拆成 **命名布尔函数** `bool valid_xxx()`，主流程可读性更高，也减少括号匹配错误。

---

## 考场 checklist（可缩印）

1. `n,m` 与循环上界是否一致？  
2. 乘法是否可能超 `int`？  
3. 多测是否清空？  
4. 输出格式与样例 **逐字符** 对比过吗？  

交卷前 60 秒只做 checklist，不改大逻辑。

---

## 小结

模拟题得分的关键是 **正确理解 + 稳健实现**。建议建立个人 **检查清单**：范围、边界、类型、样例、极端数据，在交卷前快速过一遍。更多字符串类模拟可与 `topics/string.md` 配合阅读。
