# 动态规划专题（CSP）

## 基本概念

**动态规划（DP）**用于解决多阶段决策或计数/最优化问题，通常具备：

1. **最优子结构**：整体最优解由子问题的最优解组成（最优化类 DP）。  
2. **重叠子问题**：子问题被反复求解，适合用表记录结果。  
3. **状态转移方程**：用已知更小规模状态的答案推导当前状态。

实现方式：**记忆化搜索**与**递推（循环填表）**等价时选熟练的一种；竞赛中递推常更省栈空间。

---

## 线性 DP

### 最长上升子序列（LIS）

- **状态**：`dp[i]` 表示以 `a[i]` 结尾的 LIS 长度。  
- **转移**：`dp[i] = max(dp[j] + 1)`，其中 `j < i` 且 `a[j] < a[i]`。  
- **复杂度**：O(n²)；配合二分可 O(n log n)（维护辅助数组）。

### 最长公共子序列（LCS）

- **状态**：`dp[i][j]` 表示 `A[0..i-1]` 与 `B[0..j-1]` 的 LCS 长度。  
- **转移**：  
  - 若 `A[i-1] == B[j-1]`：`dp[i][j] = dp[i-1][j-1] + 1`  
  - 否则：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`

## 背包 DP

### 01 背包

每件物品最多选一次。`dp[i][w]`：考虑前 `i` 件，容量 `w` 的最大价值。

转移（倒序循环 `w` 可压成一维）：

```cpp
// 一维滚动：for (int i = 0; i < n; ++i)
//   for (int w = W; w >= weight[i]; --w)
//     dp[w] = max(dp[w], dp[w - weight[i]] + value[i]);
```

### 完全背包

每件无限件。内层 `w` **正序**循环更新（同一件可重复用）。

### 多重背包

每件有 `cnt[i]` 个。可二进制拆分转成 01，或单调队列优化（进阶）。

---

## 区间 DP

合并区间、括号匹配、石子合并类。

- **状态**：`dp[l][r]` 表示区间 `[l, r]` 上的最优值/方案数。  
- **转移**：枚举分割点 `k`，`dp[l][r] = min/max( dp[l][k] + dp[k+1][r] + cost )`。  
- **顺序**：区间长度从小到大。

---

## 树形 DP

在树上做 DP，常从叶子向根（后序）或根向叶子。

- **状态**：`dp[u][...]` 表示以 `u` 为根的子树在某约束下的最优值。  
- **转移**：枚举子节点合并到 `u`。  

经典例：最大独立集、树的直径（也可用两次 DFS/BFS）。

---

## 状压 DP

集合用二进制表示，状态 `dp[mask][...]`，`mask` 的每一位表示某元素是否被选。

- 适用：`n` 较小（通常 `n ≤ 20` 左右视题目而定）。  
- 注意：枚举子集、超集的复杂度与剪枝。

## 优化技巧

### 滚动数组

若 `dp[i]` 只依赖 `dp[i-1]` 或固定前几层，可将第一维压成 2 行或 1 行，降低空间。

### 单调队列优化

形如 `dp[i] = min(dp[j] + cost(j, i))` 且 `j` 在一个滑动窗口内，可用单调队列维护候选最值，将 O(n²) 降为 O(n)。

## 模板骨架：01 背包一维

```cpp
#include <vector>
#include <algorithm>
using namespace std;

int knapsack01(const vector<int>& w, const vector<int>& v, int W) {
    vector<int> dp(W + 1, 0);
    int n = (int)w.size();
    for (int i = 0; i < n; ++i) {
        for (int c = W; c >= w[i]; --c) {
            dp[c] = max(dp[c], dp[c - w[i]] + v[i]);
        }
    }
    return dp[W];
}
```

## 路径还原与方案输出

若题目要求输出具体方案（选哪些物品、路径字符串），在 `dp` 表填完后 **从终态倒推**：比较转移来源取等号成立的方向，递归或循环拼出路径。注意空间：可只存「从哪来」的指针数组，避免重复 DFS。

---

## LIS 的 O(n log n) 思路（简述）

维护数组 `tail[len]`：`tail[k]` 表示长度为 `k+1` 的上升子序列的最小末尾。扫描 `a[i]` 时在 `tail` 上二分第一个 `≥ a[i]` 的位置替换；`tail` 有效长度即为 LIS 长度。适合 `n` 较大、只需长度的场景。

---

## 计数类 DP 与取模

方案数、路径数类 DP 需在每次加法后 **`%= MOD`**，减法后加 `MOD` 再 `% MOD` 防止负数。乘法用 `long long` 中转再取模。

```cpp
const int MOD = 1'000'000'007;
dp[i] = (dp[i] + dp[j]) % MOD;
```

## DAG 上的最长路 / 最短路（拓扑序 DP）

对有向无环图，将顶点按 **拓扑序** 排列后，依次松弛：`dp[v] = max/min(dp[v], dp[u] + w)`。若图可能有环，需先判环或换算法（如最短路模型）。

---

## 硬币问题辨析

- **完全背包**：每种硬币无限枚，凑成金额 **最少张数** 或 **方案数**。  
- **若硬币有固定枚数限制**：转为多重背包或暴搜剪枝（视数据范围）。  

凑方案数时注意初始化：`dp[0]=1`，其余为 0；求最小时不可简单用 `INT_MAX` 初始化后不加判断就 `+1`。

---

## 编辑距离（经典二维 DP）

`dp[i][j]`：`A[0..i-1]` 变为 `B[0..j-1]` 的最少操作次数（插入/删除/替换）。转移枚举三种操作取最小。字符串 DP 与 `topics/string.md` 可对照学习。

---

## 最大子段和（ Kadane 思想）

`dp[i]` 表示以 `i` 结尾的最大子段和：`dp[i] = max(a[i], dp[i-1] + a[i])`。可压成一变量滚动。属于最基础的线性 DP，常作为其他模型的子问题。

---

## 二维网格 DP（基础）

从左上到右下路径数/最小代价：`dp[i][j]` 由 `dp[i-1][j]` 与 `dp[i][j-1]` 转移，注意障碍格初始化无穷或 0。带 **只能向右/下/上/左** 若干步的变体需检查是否形成 DAG 或需最短路。

---

## 期望 DP（了解）

概率题中状态含期望：`E[u] = 1 + sum p * E[v]`，可能需 **方程组** 或 **拓扑逆序**。CSP 中出现频率低于省选，但见到「随机游走」关键词可建立联系。

## 常见问法与模型映射

| 题面关键词 | 可能模型 |
|------------|----------|
| 不选相邻、选数最大和 | 线性 DP / 树形 DP |
| 容量、重量、价值 | 背包 |
| 区间合并代价 | 区间 DP |
| 子集、N 小 | 状压 DP |
| DAG、依赖 | 拓扑 + DP |

## 初始化常见错误

- 求 **最小** 代价：有效状态用 `INF`，非法状态避免参与 `min` 转移。  
- 求 **最大** 代价：用 `-INF`，注意全负序列时答案仍可能为负。  
- **计数** 类：`dp[0]=1` 或按题意定义空方案。

---

## 空间限制

`n=5000` 的 `dp[5000][5000]` 的 `int` 约 100MB，可能超限；需 **滚动数组** 或换算法。开数组前估算：`元素个数 × 4 字节（int）` 与题目内存限制对比。

---

## 时间与常数

同一复杂度下 **常数** 也可能导致 TLE：减少 `memset` 全表、用 `vector` 局部代替过大静态数组、内层循环避免冗余。若 `O(n²)` 边界，注意 `n` 上限是否卡在 5000～8000。

---

## 小结

做 DP 题建议先写三句话：**状态含义、转移来源、边界初始化**。证明可用数学归纳：假设小规模正确，推导一步仍正确。与贪心的区别见 `topics/greedy.md`。
